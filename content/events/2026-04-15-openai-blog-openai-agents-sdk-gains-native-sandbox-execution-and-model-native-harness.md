## What changed

A new entry has appeared in the OpenAI News RSS feed announcing an update to the Agents SDK. The update introduces native sandbox execution, allowing agent code to run in isolated environments without requiring developers to wire up external sandboxing infrastructure. A model-native harness is also added, tightening the integration between the model and the execution layer. The update is positioned as enabling secure, long-running agents that can operate across files and tools. This is a net-new item in the feed — there was no previous content to compare against.

## Implication

For agent-infrastructure practitioners, native sandbox execution reduces the engineering overhead and security risk of running untrusted or semi-autonomous code generated by agents. The model-native harness signals that OpenAI is standardizing how models interact with tool-calling and file I/O loops at the SDK level, which may shift best practices away from custom orchestration layers. Developers building on the Agents SDK should review whether their existing sandboxing and execution logic is now redundant or can be simplified. Publishers and platform teams evaluating agentic workflows should note this lowers the barrier to deploying long-running agents, but also means tighter coupling to OpenAI's own execution abstractions. This continues OpenAI's pattern of vertically integrating agent infrastructure to reduce friction while increasing platform dependency.
