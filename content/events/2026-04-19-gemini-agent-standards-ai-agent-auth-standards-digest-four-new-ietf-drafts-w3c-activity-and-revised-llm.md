## What changed

The updated digest adds four entirely new IETF drafts not present in the previous version: Web Bot Auth (WBA) for cryptographic HTTP request signing with Google and Cloudflare involvement; draft-drake-agent-identity-registry-00 proposing a federated, hardware-anchored OIDC identity registry; draft-sharif-apki-agent-pki-00 defining an Agent PKI for ephemeral AI agent lifecycles; and draft-oauth-transaction-tokens-for-agents-06 extending OAuth Transaction Tokens with agent context propagation via the 'act' field. A W3C AI Agent Protocol Community Group white paper (April 1, 2026) on inter-agent authentication is also newly surfaced. Two items from the prior digest are dropped: the Agent Identity Protocol (draft-aip-agent-identity-protocol-00) and the Anonymous Bot Authentication draft (draft-rescorla-anonymous-webbotauth-00), as well as the Auth0/Akamai GA announcement and the agents-brief.txt community evolution story. Notably, the llms.txt assessment reverses direction — previously described as having "very limited adoption" with LLM bots ignoring it, it is now characterized as gaining traction as a de facto standard with rapidly increasing developer and SEO adoption.

## Implication

The surge to at least five active IETF drafts on agent identity and authentication in a single 30-day window signals that the standards layer for agentic infrastructure is moving from exploratory to competitive, with multiple incompatible approaches (PKI, registry, token extension, cryptographic signing) now in parallel development — implementors should track convergence or fragmentation risk carefully. Google's active experimentation with WBA and Cloudflare's contributor role are the most consequential near-term signals for publishers and CDN operators, since WBA would require web servers to verify cryptographic signatures from legitimate crawlers, directly affecting bot-filtering and access-control infrastructure. The contradictory llms.txt narrative — previously "ignored by LLM bots," now "rapidly growing de facto standard" — reflects genuine uncertainty in the ecosystem and warrants skepticism; publishers should not treat either characterization as settled. The agents.txt IETF draft expiring without adoption, combined with the absence of a successor, leaves agent capability discovery without a standards-body home, increasing the likelihood that proprietary or de facto mechanisms (MCP endpoints, llms.txt, WBA) fill the gap. Agent-infrastructure practitioners should begin evaluating whether their toolchains can handle OAuth Transaction Token 'act' field propagation, as draft-06 suggests this pattern is maturing toward implementation readiness.
