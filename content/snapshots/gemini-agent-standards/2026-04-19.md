Here's a compact digest of new or updated IETF drafts, W3C proposals, and standards-body activity on AI agent authentication, web bot auth, agents.txt, llms.txt, or agent identity from the last 30 days:

1.  **IETF Draft: AI Agent Authentication and Authorization (draft-klrc-aiagent-auth-01)**
    This Internet-Draft, updated on March 30, 2026, proposes a model for authenticating and authorizing AI agent interactions by composing existing standards like Workload Identity in Multi-System Environments (WIMSE) and OAuth 2.0. It aims to consolidate current practices and identify areas where new standardization efforts are needed for agent-based systems.
    Source: [https://datatracker.ietf.org/doc/draft-klrc-aiagent-auth/](https://datatracker.ietf.org/doc/draft-klrc-aiagent-auth/)

2.  **IETF Draft: Anonymous Bot Authentication: Authorization and Rate Limiting for Web Agents (draft-rescorla-anonymous-webbotauth-00)**
    Published on April 8, 2026, this Internet-Draft introduces Anonymous Bot Authentication (ABA), a system designed to help website operators differentiate between desired and undesired bot traffic. ABA allows for rate limiting and access control for automated agents without requiring the linking of individual requests to a specific sender.
    Source: [https://datatracker.ietf.org/doc/draft-rescorla-anonymous-webbotauth/](https://datatracker.ietf.org/doc/draft-rescorla-anonymous-webbotauth/)

3.  **IETF Draft: Agent Identity Protocol (AIP): Decentralized Identity and Delegation for AI Agents (draft-singla-agent-identity-protocol-01)**
    This Internet-Draft was submitted on April 17, 2026. It proposes an Agent Identity Protocol (AIP) focused on decentralized identity and delegation mechanisms specifically for AI agents.
    Source: [https://datatracker.ietf.org/doc/draft-singla-agent-identity-protocol/](https://datatracker.ietf.org/doc/draft-singla-agent-identity-protocol/)

4.  **W3C Community Group Launch: Agentic Integrity Verification Specification Community Group**
    Launched on April 5, 2026, this W3C Community Group aims to develop open formats for cryptographic proof of AI agent sessions. The goal is to create portable, self-verifiable records of agent actions, which can be independently verified without external infrastructure, addressing the need for auditability in AI systems.
    Source: [https://www.w3.org/community/aivs/](https://www.w3.org/community/aivs/)

5.  **Proposal Update: agent-manifest.txt (formerly agents.txt)**
    In March 2026, the `agents.txt` proposal was renamed to `agent-manifest.txt` to better reflect its purpose as a rich capability manifest for AI agents. This proposed web standard aims to provide a structured way for websites to declare what AI agents can do, how to interact with them (e.g., preferred APIs, authentication methods), and under what terms, similar to how `robots.txt` guides web crawlers.
    Source: [https://dev.to/jasper/agent-manifesttxt-a-proposed-web-standard-for-ai-agents-formerly-agentstxt-2026-403h](https://dev.to/jasper/agent-manifesttxt-a-proposed-web-standard-for-ai-agents-formerly-agentstxt-2026-403h)
