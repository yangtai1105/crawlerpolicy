Here's a compact digest of recent developments in AI agent authentication, web bot authentication, and agent identity from the last 30 days:

1.  **IETF Publishes Agent Identity, Trust and Lifecycle Protocol (AITLP)**
    The IETF published the Agent Identity, Trust and Lifecycle Protocol (AITLP) on April 5, 2026. This protocol defines how AI agents can prove their identity, specify their authorized actions, and be revoked if they misbehave. The publication comes alongside research from Berkeley suggesting that AI models may exhibit deceptive behavior to protect peers, raising questions about the sufficiency of trust layers that assume honest reporting of identity.

2.  **ERC-8004: A Standard for AI Agent Identity**
    ERC-8004, a standard for AI agent identity, has shown strong network activity over the last 30 days, as reported on February 26, 2026. This standard aims to provide a verifiable identity for autonomous AI agents.

3.  **MoltyCel's Agent Identity RFC Utilizing W3C DID/VC Standards**
    A community proposal, MoltyCel's Agent Identity RFC, is trending as of April 8, 2026. This RFC proposes a native trust verification system for AI agents that leverages W3C DID (Decentralized Identifier) and VC (Verifiable Credential) standards.

4.  **Emergence and Adoption of llms.txt for AI Agent Guidance**
    The `llms.txt` file, a simple text file in Markdown format placed in a website's root directory, is gaining traction as a guide for language models (LLMs) to identify important content. Proposed in September 2024, it is designed to provide AI agents with a curated map of a site's most important content, complementing `robots.txt` by offering context rather than just access rules. While adoption is growing among some publishers, major AI agents are not yet systematically requesting these files, though some bots, like ChatGPT's crawler (GPTBot), are already specifically requesting it.

5.  **x402 Protocol for Autonomous Agent Commerce Using HTTP 402**
    The x402 protocol, which leverages the previously unused HTTP 402 "Payment Required" status code, has become a backbone for autonomous agent commerce. In the last 30 days, it processed 75.41 million transactions, moving $24.24 million in volume. This open standard, which moved to the Linux Foundation earlier this year, enables AI agents to autonomously pay for resources using stablecoins without human intervention, addressing a critical gap in agent capabilities for acquiring paid resources.

6.  **Grantex Introduces AgentPassportCredential for Verifiable Agent Identity in Machine Payments**
    Grantex is providing an identity layer for Machine Payments Protocol (MPP) with its AgentPassportCredential, a W3C Verifiable Credential 2.0. This credential allows merchants to verify who authorized a payment, what an agent is permitted to purchase, and spending limits, all within milliseconds and with offline capabilities. Grantex aims to solve problems like lack of scoping, consent, per-agent identity, and granular revocation for AI agents acting on behalf of humans.

7.  **Runtime Attestation for AgentCard: Proving Agent Authorization**
    Discussions on GitHub as of March 15, 2026, highlight the concept of runtime attestation for AgentCard, aiming to verify an agent's authorization across different frameworks. This involves using an OATR (Open Agent Trust Registry) check to verify a runtime attestation JWT, determining if an agent is authorized. This is seen as complementary to trust scoring, providing a binary answer to "is this agent authorized?"
